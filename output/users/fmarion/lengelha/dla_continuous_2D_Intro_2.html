  2. Extend this DLA  simulatlion to three dimensions. (In the View, you will need to use a DrawingFrame3D obect.) How does the fractal dimension of the three-dimensional structures compare to that of the two-dimensional structures?
  3. As it is written, the program should be able to create clusters of >10,000 particles fairly quickly. However, as the number of particles becomes large, the simulation will eventually become very slow. This is because every step of the random walk involves a collision-detection calculation for every particle in the cluster. Hence, this is an O(n) calculation. (See this link.) In one dimension, a "binary search tree" can be used to store an ordered list of numbers that can be searched very efficiently. The two-dimensional analog of a binary search tree is called a "quadtree". To improve the efficiency of the DLA  simulation, implement a quad-tree. This will greatly reduce the number of collision-detection calculations that are required for large clusters (although it will take some programming!).
  4. Improve the efficiency of your three-dimensional DLA  simulation by implementing an octtree, which is the three-dimensional analog of a binary search tree.
