We first express the system of equations as a single matrix equation,

Mx=b(1)(1)Mx=b

Since there are as many equations (rows in MM) as there are variables (columns in MM), MM will be a square matrix. If there is a solution to this set of equations, then there will exist another matrix M−1M−1 such that M−1M=IM−1M=I, where II is the identity matrix. In that case,

M−1Mx=M−1b(2)(2)M−1Mx=M−1b
Ix=x=M−1b(3)(3)Ix=x=M−1b

There are algorithmic methods for finding M−1M−1 (and thus xx; in our case we’ll be using standard libraries to calculate this matrix and the corresponding solution. One standard method is to have the computer calculate M−1M−1 directly using an “invert()” function, and then multiply that inverse matrix by bb to obtain xx.

Most computational packages that have the ability to calculate M−1M−1 also have an additional function, often called “solve()”, that takes care of the multiplication as well, and gives the user xx directly.

For more information about how these functions work, see Numerical Recipes in C by Flannery, Teukolsky, Press, and Vetterling, or similar text.
